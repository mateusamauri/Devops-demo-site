Docker

Docker é uma plataforma para desenvolver, enviar e executar aplicações em containers. Ele oferece isolamento, portabilidade e consistência entre ambientes de desenvolvimento, teste e produção.

Imagem vs Container

Imagem: snapshot imutável com código e dependências.
Container: execução viva da imagem.

Dockerfile

  • FROM — imagem base
  • RUN — executa comandos
  • COPY — copia arquivos
  • CMD — comando de inicialização

Principais comandos

  • docker build -t minha-img .
  • docker run -p 80:80 minha-img
  • docker exec -it id bash

Kubernetes

Kubernetes é um orquestrador de containers. Ele gerencia o deploy, escala e manutenção de aplicações containerizadas.

  • Pod: menor unidade, contém container(es)
  • Deployment: define número de réplicas
  • Service: expõe o app internamente ou externamente
  • YAML: arquivos de configuração

Comandos kubectl

  • kubectl apply -f arquivo.yaml
  • kubectl get pods
  • kubectl logs pod-name

Terraform

Terraform é uma ferramenta de IaC (Infraestrutura como Código). Permite criar e gerenciar recursos em nuvem com arquivos declarativos.

  • init — inicializa o projeto
  • plan — mostra as mudanças
  • apply — executa o plano

Exemplo de recurso para criar uma EC2:

resource "aws_instance" "web" {
  ami = "ami-id"
  instance_type = "t2.micro"
}

AWS

AWS (Amazon Web Services) é uma plataforma de serviços em nuvem.

  • EC2: máquinas virtuais
  • S3: armazenamento de objetos
  • IAM: controle de acesso
  • CloudWatch: logs e métricas

Configuração CLI

aws configure

Fornece Access Key, Secret Key e região padrão.

CI/CD

CI (Integração Contínua) executa testes a cada push.
CD (Entrega Contínua) automatiza o deploy.

  • Ferramentas: GitHub Actions, GitLab CI, Jenkins
  • Pipelines: sequência de etapas automatizadas

Exemplo de pipeline GitHub Actions:

name: Deploy
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - run: docker build -t app .

Git

Git é um sistema de controle de versão distribuído.

  • git clone, git commit, git push
  • git branch, git merge
  • Trabalho com branches: feature, develop, main

Linux

Linux é um sistema operacional usado amplamente em servidores e ambientes de nuvem.

  • ls, chmod, tail, grep
  • Permissões: leitura, escrita, execução (rwx)

Redes

Conceitos básicos essenciais em DevOps:

  • DNS: resolve nomes em IP
  • HTTP vs HTTPS: protocolo + segurança via TLS/SSL
  • Portas: 80 (HTTP), 443 (HTTPS), 22 (SSH)

Monitoramento

Ferramentas e práticas para garantir visibilidade e saúde da aplicação.

  • Logs: CloudWatch, ELK
  • Métricas: Prometheus + Grafana
  • Alertas: thresholds, webhooks, e-mail
